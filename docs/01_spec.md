01_spec.md: 要件定義・システム仕様書

1. プロジェクト概要

給料日を起点とした固定費の支払管理（ToDo形式）と、銀行口座残高の連動を行う家計管理アプリケーション「BudgetFlow」。

2. ターゲットユーザー

月々の固定費支払いを確実に管理したい個人。

支払日ごとにどの口座からいくら引かれるかを可視化したいユーザー。

3. 主要機能詳細

A. 認証・ユーザー基盤

Firebase Auth によるユーザー分離。匿名認証から開始し、必要に応じて永続化。

B. 口座管理 (Assets)

複数口座の登録・編集。

各口座の現在残高の保持。

支払実行時の残高自動減算。

C. テンプレート管理 (Definitions)

固定費のマスター登録（名称、標準予算、支払予定日、引き落とし口座）。

表示順序のカスタマイズ。

D. 期間サイクル管理 (Cycle)

「今月を開始」：テンプレートから当月分のToDoを生成。

「サイクル完了」：当月の支払実績を履歴へ移動し、ToDoをリセット。

E. 支払ToDo操作 (Execution)

支払実行：実支払額の入力、ステータス更新、口座残高反映（トランザクション）。

支払取消：ステータスを未払いに戻し、口座残高を復元（逆トランザクション）。

F. データ可視化・履歴 (Analytics)

過去のサイクル結果の閲覧。

月別の支出推移グラフ。

4. 非機能要件

UX: モバイルでの操作性を重視したUI。角丸を多用したモダンなデザイン。

データ整合性: Firestoreトランザクションによる残高とステータスの一貫性確保。
