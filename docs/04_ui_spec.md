04_ui_spec.md: UI・インタラクション詳細仕様

1. ログイン画面 (Login View)

メインボタン: 「ログイン」 (匿名認証の実行)

新規誘導: 「アカウントをお持ちでない方はこちら」セクションを設け、「新規登録してはじめる」ボタンを配置。

補足説明: 匿名ログインの性質（ブラウザ保存）に関する注意書きを表示。

2. 初期セットアップ (Zero State)

口座未登録時:

ダッシュボードの口座エリアに「口座を追加」ボタンを強調表示。

ToDoエリアに「口座を登録すると、固定費の設定ができます」という案内を表示。

テンプレート未登録時:

「固定費テンプレートを作成」ボタンを表示し、モーダルへ誘導。

3. 口座管理モーダル (Account Modal)

入力項目:

名前 (string)

現在の残高 (number)

アイコン選択 (Material Symbols から選択)

インタラクション:

保存時に Firestore トランザクションは不要（単一ドキュメント更新のため）。

削除時は「この口座に紐づいているテンプレート・ToDoがある場合」に警告を表示。

4. 固定費テンプレート管理 (Template Management)

入力項目:

項目名 (例: Netflix, 家賃)

予算 (number)

支払予定日 (1-31の数値)

引き落とし口座 (登録済み口座からセレクトボックスで選択)

アーカイブ仕様:

テンプレートを削除する際、物理削除ではなく isArchived: true に設定。

これにより、過去の履歴（Record）における参照整合性を維持しつつ、新規ToDo生成対象から除外する。

5. 並び替え (Reordering)

モード切替: 「AUTO (支払日順)」と「CUSTOM (ユーザー指定順)」をトグルで切替。

保存: CUSTOMモードでのドラッグ完了時に、全項目の order フィールドを Batch Update する。
